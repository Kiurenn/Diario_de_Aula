## Agregação de Valor

Educação, trabalho, entre outras coisas, não necessariamente entregará o resultado esperado.

A realização algo deve ser dividida em etapas.
- Planejamento e Estruturação: não deve ser em excesso pois atrapalha e consome muito tempo;
- Execução é a realização do projeto;
- Implantar é por para utilização;
- Verificação é identificar se está correta a execução de uma estápa
- Validação consiste em identificar se algo foi compreendido;

Resultado é atigir os requisitos desejados, entregandos valores agregados juntamente deles para maximizar a sastifação de quem espera por um resultado.

## Atividade em Equipe

FASES DO DESENVOLVIMENTO
ESTRUTURAÇÃO
- REQUISITOS -> ANÁLISE -> PROJETO
EXECUÇÃO
- CODIFICAÇÃO -> TESTES
FINALIZAÇÃO
- IMPLEMENTAÇÃO -> MANUTENÇÃO

- Modelo Cascata
  - A equipe só avança para a próxima etapa a partir da conclusão da atual. É um modelo pouco flexível, já que o processo fica estagnado caso algo não seja
desenvolvido como desejado, não dando opções de prosseguir com outras etapas enquanto aquela não for finalizada.
- Modelo BigBang
  - O desenvolvimento do projeto inicia a partir do momento que é recebido, sem realizar um planejamento, apenas executando como pensam que deve ser feito. Este modelo
apresenta vários riscos no seu processo de desenvolvimento, visto que não há uma divisão de etapas definidas em um planejamento.
- Model Espiral
  - O modelo espiral consiste na criação de versões do produto final a cada volta, em que cada volta contém cerca 4 etapas a serem cumpridas para prosseguir
para a próxima volta. A cada volta, é evoluída a versão do sistema para chegar mais próximo da versão final.

Ideologias do metodo agil
- Pessoas que fazem o software executar
- Entender o que o cliente deseja
- Software Funcionando
- Aceitar mudanças

Exemplos de Metodos Ageis
- XP: Método completo porém engessado
- Scrum: Framework (foco na entrega de valor)
- Kanban: Gestão de Tarefas
